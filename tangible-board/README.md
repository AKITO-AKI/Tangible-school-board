# Tangible School Board — サイト&デモ ドキュメント

このリポジトリは「Tangible School Board」の紹介サイト（`site/`）とブラウザで動くデモ（`board.html`）を含みます。見る人（生徒・先生・保護者）にも、作る人（エンジニア・デザイナ）にも伝わるよう、内容面と技術面の両方をまとめました。

## コンセプト（内容的な側面）

- 問い：学校の連絡は、なぜ“見落とし”や“抜け”が起きるのか？
- 仮説：連絡を「スマホの中」だけに閉じず、廊下や昇降口など“みんなが通る場所”に置くボードがあれば、情報は届きやすくなる。
- 解：カテゴリ（全体/部活動/委員会/学年）、カレンダー、短いティッカーを一枚に統合し「立ち止まった数秒で今日が分かる」体験を設計。
- タンジブル（手触り）：身体を動かして“そこにある”場所へ行く。視線の重心をスマホから“場”へ移す。情報の説得力を環境側で補強する。

### サイトの流れ

- メイン（`site/index.html`）：Apple風の白基調で導入。ロール切替（生徒/先生/保護者）で“語り口”を出し分け。
- 機能紹介（`site/features.html`）：Dayflow（1日のどこで効くか）／UIハイライト同期／状態付きデモジャンプ。
- なぜ（`site/why.html`）：課題・データゾーン／情報密度ヒートマップ（スクラブ）。
- タンジブル（`site/tangible.html`）：距離スライダーで見え方を変える。背景シーンのクロスフェード。
- 今後（`site/roadmap.html`）：展望・課題。
- 制作（`site/making.html`）：制作ストーリー／「どこに置く？」ミニ質問。

### 触れる要素の例

- ロール切替：読み手の立場をボタンで切替、関連テキストを即時出し分け。
- Dayflow：登校/ホームルーム/放課後/テスト前の各シーンでボードが“働く瞬間”をプレビュー。
- UIハイライト同期：説明文に合わせてモック上の注目領域がハイライト。
- 状態付きジャンプ：紹介カードからクエリ付きでデモを起動（例 `board.html?category=club&date=2025-06-12`）。
- 距離スライダー：近/中/遠で縮尺・注目度が変わる見え方を体験。
- ヒートマップ：月内の情報集中度を視覚化。スクラブで該当日をプレビュー。
- 置き場所ミニ質問：昇降口/階段ホール/職員室前/その他から選択、説明と画像が切替（保存）。

## 技術構成（技術的な側面）

- スタック：静的 HTML/CSS/JavaScript（vanilla）。ビルド不要。
- サイト側構造：`site/` 配下にページ別 HTML。共通スタイル `site/common.css`、共通挙動 `site/common.js`。
- デモ側：`board.html` + `styles.css` + `script.js`。初回チュートリアル（スワイプヒント）あり。

### 共通CSS（`site/common.css`）

- デザインシステム：白基調・シャドウ・余白・フォーカスリング。
- コンポーネント：カード、グリッド、タイムライン、進捗インジケーター。
- 追加スタイル：ロールトグル、Dayflow、UIホットスポット、距離スライダー、ヒートマップ、配置質問。

### 共通JS（`site/common.js`）

- Reveal演出：IntersectionObserverで`.reveal`要素にフェードイン。
- ナビ状態：カレントページを`.menu a`に反映。
- サイドドット：ページ遷移（`data-href`）。
- 役割トグル：`.role-btn`と`[data-role-text]`でテキスト出し分け（localStorage保存）。
- Dayflow：`.dayflow-step`クリックで画像/キャプション更新（軽いフェード）。
- UI同期：`.ui-feature-item`と`.ui-hotspot`のアクティブ切替（hover/IntersectionObserver）。
- デモジャンプ：`.feature-demo-link`をクリックで`board.html?`クエリを生成し新規タブを開く。
- 距離スライダー：`#distance-range`で`data-distance`を`near/mid/far`に切替、説明更新。
- ヒートマップ：セルを生成し、mousemoveスクラブで該当日をプレビュー。
- 置き場所質問：`.placement-option`で画像と文言を更新（localStorage保存）。

### デモJS（`script.js`の拡張余地）

- クエリ読取：`category`/`date` を読んで初期状態反映（未連携なら今後追加）。
- 初回チュートリアル：`board.html`内の`[data-swipe-hint]`をlocalStorage制御。

## 開発者向けガイド

- ファイル追加：ページを増やす場合は`site/xxx.html`を作り、`common.css`/`common.js`をlink/script。
- コンポーネント再利用：HTMLのdata属性（例：`data-role-text`/`data-feature`）を踏襲すると共通JSが動きます。
- アセット：`assets/`以下に画像を配置。ホットスポットやDayflow画像は差し替え可能。
- モーション配慮：`prefers-reduced-motion`を尊重。長いアニメは控えめに。
- アクセシビリティ：`role`/`aria-*`を適宜付与、キーボード操作を担保。

## 使い方（閲覧者向け）

1. `site/index.html` をブラウザで開く
2. ロール（生徒/先生/保護者）を切り替えて読み比べ
3. 機能紹介ページでDayflowやUI同期を操作
4. 「この状態のデモを見る」からデモを起動
5. タンジブルの距離スライダーやヒートマップ、配置質問でイメージを深める

## ローカルでの動かし方（Windows PowerShell）

- そのままHTMLをダブルクリックで開けます。CORS制約を避けたい・複数ページの遷移を楽にする場合は、簡易HTTPサーバーを使う方法もあります。

```powershell
# Pythonがある場合（推奨）
python -m http.server 8000
# ブラウザで http://localhost:8000/site/index.html を開く
```

> 注: 画像アセット（`assets/...`）はダミーの参照が含まれます。必要に応じて実画像を追加してください。
